Remarks
ATTENTION: An application should have at least one receiver callback registered per receiver object. Use of this constructor creates a window after receiver creation and before adding a callback in which messages may be lost; therefore, its using this constructor is not recommended. Use a constructor that specifies at least one initial callback instead. Warning: It is not safe to call this constructor from a context thread callback.
